## 1. User journey

### Daily (post-hackathon)

1. **8am UTC cron** triggers W6′ Inngest function.
2. Function synthesizes ~200-word brief з yesterday's runs + signals + drafts.
3. Slack incoming webhook posts message до `SLACK_WEBHOOK_URL` channel.
4. CMO opens Slack коли drinking coffee — sees brief з deep links to dashboard.

### Demo (today)

1. User clicks **"Send today's brief now"** button у dashboard footer (W6′ section).
2. Server action emits `morning-brief.tick` event.
3. Within 30s — Slack message posts до demo channel.
4. UI shows live "Sent ✓" status з timestamp + link до Slack message permalink.

## 2. Brief content template

Generated by LLM (gpt-4o-mini, ~$0.005 per call) з prompt that constrains:

```
Heading (line 1): "📊 BBH morning brief — {date}"

Section 1 — TL;DR (2-3 sentences):
One-line summary of yesterday: signal count, severity breakdown, drafts ready.

Section 2 — Top signals (max 3 bullets, severity high first):
• [{severity icon}] {brand} — {1-line summary}. {citation source}.

Section 3 — Drafts pending review:
{N} drafts waiting. {top 1 mentioned by channel hint}. [Open dashboard →]

Section 4 — Brand pulse (Peec snapshot):
Visibility: {N}% {direction-arrow} vs prior day.
Avg position: {N} {direction-arrow}.
Sentiment mix: {pos/neutral/neg %} {direction-arrow}.

Section 5 — CTA:
Approve drafts via dashboard → {dashboard_url}
```

LLM constrained до ≤200 words. Markdown format (Slack-flavored — `*bold*`, `_italic_`, links як `<URL|text>`).

Empty state (no signals/drafts yesterday): "Quiet day — no high-severity signals. Brand pulse stable. ☕ Drink your coffee."

## 3. Inputs

Aggregated SQL query на yesterday's data (last 24h ending at 00:00 UTC today):

```sql
-- yesterday's signals
select severity, sentiment, summary, source_url, c.display_name as competitor
from signals s
left join competitors c on s.competitor_id = c.id
where s.organization_id = $1
  and s.created_at >= now() - interval '24h'
order by severity desc, s.created_at desc;

-- yesterday's drafts
select status, body, channel_hint
from counter_drafts
where organization_id = $1
  and created_at >= now() - interval '24h';

-- latest Peec snapshot delta
select stats from runs
where organization_id = $1
  and function_name = 'competitor-radar'
  and ok = true
  and finished_at >= now() - interval '24h'
order by finished_at desc
limit 1;
```

## 4. Outputs (per `MorningBriefSchema`)

```ts
// lib/schemas/morning-brief.ts (CONTRACTS.md §2.10)
export const MorningBriefSchema = z.object({
  delivery_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),    // YYYY-MM-DD
  summary_body: z.string().min(50).max(2000),                 // markdown-formatted brief
  signal_count: z.number().int().nonnegative(),
  severity_breakdown: z.object({
    high: z.number().int().nonnegative(),
    med: z.number().int().nonnegative(),
    low: z.number().int().nonnegative(),
  }),
  drafts_pending: z.number().int().nonnegative(),
  brand_pulse: z.object({
    visibility_pct: z.number().nullable(),
    avg_position: z.number().nullable(),
    sentiment_mix: z.object({
      positive_pct: z.number(),
      neutral_pct: z.number(),
      negative_pct: z.number(),
    }).nullable(),
  }).nullable(),                                              // null якщо no Peec data yet
  evidence_refs: z.array(z.string()).min(1),                  // signal IDs + URLs
});
```

## 5. Pipeline (W6′)

`inngest/functions/morning-brief.ts`:

1. **Step 1 `gather-yesterday-data`** — execute aggregation SQL.
2. **Step 2 `synthesize-brief`** — `generateObject({schema: MorningBriefSchema})` з aggregated context.
3. **Step 3 `format-slack-blocks`** — convert markdown body до Slack Block Kit format (better rendering than plain markdown).
4. **Step 4 `send-slack`** — POST to `SLACK_WEBHOOK_URL` з blocks payload. Catch errors.
5. **Step 5 `persist-delivery`** — INSERT row у `brief_deliveries`:
   - `status='sent'` якщо webhook 200, else `'failed'` з `error_reason`.
   - `recipient` = ENV`SLACK_DEMO_CHANNEL` (e.g. "#bbh-demo").
6. **Step 6 `persist-run`** — runs row з stats.

Cost envelope: ~$0.005 per run (single short LLM call) + Slack webhook free.

Cron schedule (post-hackathon): `0 8 * * *` UTC (8:00 UTC daily). Per-org timezone — post-hackathon (`organizations.local_timezone` already у schema).

## 6. Slack integration

- **Webhook URL:** `SLACK_WEBHOOK_URL` env var. User creates incoming webhook у workspace settings → app receives URL.
- **Message format:** Slack Block Kit (richer than plain text):
  - Header block (date + emoji).
  - Section block (TL;DR).
  - Divider.
  - List blocks (signals, drafts).
  - Action block (button "Open dashboard" з URL).
- **Errors handled:**
  - Webhook 4xx → log + `status='failed'` + `error_reason`.
  - Webhook 5xx → Inngest auto-retry (3 attempts з exponential backoff).
  - Rate limit (429) → retry з longer delay.

## 8. Acceptance criteria

### Functional

- [ ] Cron triggers manually emittable event (для demo).
- [ ] Brief generated correctly з aggregation SQL + LLM call.
- [ ] Slack webhook receives Block Kit payload.
- [ ] Slack message renders з proper formatting (header, sections, button).
- [ ] `brief_deliveries` row INSERT'ться з status.
- [ ] Dashboard "Send today's brief now" button works for live demo.
- [ ] Empty state graceful (no signals → "quiet day" message).

### Non-functional

- [ ] LLM call wrapped у `step.run()`.
- [ ] Slack send wrapped у `step.run()` для retry.
- [ ] Rate limits handled (429 → exponential backoff).
- [ ] Cost у `runs.stats.cost_usd_cents`.
- [ ] No PII у Slack payload (тільки публічні brand mentions).
- [ ] Brief body ≤2000 chars (Slack message limit).
